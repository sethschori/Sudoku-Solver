Sudoku Solver

Sudoku Solver solves Sudoku puzzles using two approaches: 1. trial-and-error computer techniques (recursive) and 2. human-style techniques (process of elimination). Please note that approach #2 only solves relatively easy Sudoku puzzles, as it employs fewer techniques than human players use. As far as I know, approach #1 can solve any Sudoku puzzle.

About Sudoku Solver

Why two approaches? While I was still relatively new to learning programming and JavaScript, I decided to try and build a Sudoku-solving program. At the time, I didn't understand recursion very well, and using human techniques for solving Sudoku seemed like the best approach. I struggled with adding different types of human techniques (see below for descriptions of Sole Candidate, Unique Candidate, and Naked Subset). Eventually, as I came to understand recursion better, I realized that it's much easier for computer programs to solve Sudoku using a recursive trial-and-error approach. Once I tried the recursive approach, I was able to write it fairly quickly and with about half the lines of code that I'd used for the process of elimination approach. As it currently stands, the elimination approach can only solve easy- and medium-level puzzles (as categorized by websudoku.com). If I increased the number of elimination techniques (as used by human players), the program would likely solve all Sudoku puzzles. However, I've stopped building this program, as I've accomplished my goals with this project and want to try other things.

How the recursive approach works: With the recursive approach, Sudoku Solver solves Sudoku puzzles by recursively testing multiple possibilities, until it finally lands on the solution. This approach violates the rules that human players are supposed to follow, although I found it much easier to code the recursive approach than the process of elimination approach. To solve the board recursively, the program finds the first blank cell on the board and starts by inserting the number 1 into that cell. It then checks to see whether there is already a number 1 in the row, column, and nonet which contain that blank cell. If there is already a number 1, then the program moves on successively to 2, 3, etc., all the way up to 9. Once it finds a number that doesn't conflict with the associated row, column, and nonet, the program moves on to the next blank cell. The process repeats. If the program cannot find a non-conflicting candidate for the subsequent blank cell, then the program goes back to the previous cell and increments to the next candidate number (e.g. it moves up from 2 to 3). If the program does find a non-conflicting candidate for the subsequent blank cell, then the process continues in the same fashion until there are no more blank cells left to fill.

How the process of elimination approach works: With the process of elimination approach, Sudoku Solver solves Sudoku puzzles by following the same rules that human players must follow (i.e. it doesn't use trial-and-error to test multiple possibilities, but instead uses rules of logic). The reasons for employing this elimination approach are 1) it enabled me learn how to translate rules of play into JavaScript functions and 2) in the future this program could be evolved to demonstrate to the user how it solves the board.

I am grateful to kristanix.com for the Suduko-solving techniques which I have used in the process of elimination approach, and I have used the techniques' names as the names of functions in the sudokusolver-elimination JavaScript program. Sudoku Solver uses three techniques from kristanix.com to solve Sudoku puzzles: Sole Candidate, Unique Candidate, and Naked Subset.

  • Sole Candidate: Given a blank cell on the board, check whether eight values are already present in the nonet, row, and column. If so, the "missing" ninth value must be the solution to that cell.
  • Unique Candidate: Check whether there is a number which -- by virtue of being in other columns and rows that intersect the other blank cells in the target cell's nonet -- can only be valid in that cell. If so, that must be the solution to that cell.
  • Naked Subset: This approach goes through the board, one row at a time, and determines all the potential values which could be filled in for each of the blank cells. Next, it checks all of the values for that row and determines if there are the same two potential values (e.g. 1, 4) which appear for two blank cells. If so, that means that those two values (1, 4) must be used in those two cells and can't appear in another cell in that row. Therefore, if there are other cells in that row whose potential values include that pair of values (e.g. 1, 2, 4, 7), it removes the pair of values from those cells (e.g. it would remove 1, 4, which would leave 2, 7 as the remaining options). As it removes potential values from each cell, the program checks whether there are any cells which have one value left, and if so that one must be the solution for that cell. Once all rows have been checked, then the process is repeated for all of the columns.
